# 玩具出售监管平台

## 一、项目简介

玩具出售监管平台是一个面向玩具销售、平台监管和消费者查询的全栈 Web 应用系统。系统围绕玩具商品上架、资质审核、销售监管、违规处理、数据统计等业务展开，支持前台用户访问和后台管理监管。

平台适用于玩具电商监管、校园周边玩具销售管理、区域市场监管模拟系统等场景。

## 四、核心特点

1. **前后台分离**
   - 前台面向普通用户和商家
   - 后台面向监管人员和系统管理员

2. **多角色权限控制**
   - 不同角色拥有不同操作权限
   - 支持管理员、监管人员、商家、普通用户等身份

3. **商品审核监管流程**
   - 商家发布商品后需监管人员审核
   - 审核通过后商品才可公开展示

4. **安全认证信息管理**
   - 商品需绑定检测报告、安全认证编号等信息
   - 用户可查询商品安全状态

5. **投诉与违规处理机制**
   - 用户可举报问题商品或商家
   - 监管人员可处理投诉并进行下架、警告等操作

6. **Docker 部署**
   - 前端、后端、数据库、缓存等服务通过 Docker 编排部署
   - 方便系统迁移、演示和上线

## 七、系统难点

1. **多角色权限控制**
   - 需要区分普通用户、商家、监管人员、管理员的权限边界。

2. **商品审核流程**
   - 商品从提交、审核、驳回、整改、通过到上架，需要完整状态流转。

3. **投诉处理流程**
   - 投诉信息需要关联用户、商品、商家和监管处理记录。

4. **文件上传与管理**
   - 商品图片、商家资质、检测报告等文件需要分类存储和访问控制。

5. **数据统计**
   - 后台需要统计商品数量、投诉数量、违规商家数量、审核通过率等数据。

6. **Docker 一体化部署**
   - 需要将前端、后端、数据库、Redis、Nginx 等服务统一编排。

## 九、Docker 部署规划

系统使用 Docker Compose 进行统一部署，主要包含以下服务：

- `frontend`：前端 Vue 项目构建后的静态资源服务
- `backend`：Spring Boot 后端服务
- `mysql`：MySQL 数据库服务
- `redis`：Redis 缓存服务
- `nginx`：反向代理服务
- `minio`：文件存储服务，可选

部署后访问地址示例：

- 前台地址：http://localhost
- 后台地址：http://localhost/admin
- 后端接口：http://localhost/api
- 接口文档：http://localhost/doc.html

## 十一、当前代码实现

本仓库已按上述方案生成一个可运行的全栈演示系统：

- `backend`：Spring Boot REST API，包含登录、商品查询、商家审核、商品审核、投诉处理、统计面板等接口
- `frontend`：Vue 3 + TypeScript + Element Plus 前端，包含前台查询页、登录页、监管后台页
- `deploy`：Nginx 反向代理配置与 MySQL 初始化脚本
- `docker-compose.yml`：统一编排前端、后端、MySQL、Redis、Nginx

说明：当前后端使用内存种子数据实现核心监管流程，同时保留 MySQL 初始化脚本，便于后续替换为 MyBatis-Plus 持久化版本。

## 十二、运行方式

### Docker Compose 部署

```bash
docker compose up --build
```

访问地址：

- 前台：http://localhost
- 后台：http://localhost/admin
- 后端接口：http://localhost/api
- Swagger：http://localhost/api/swagger-ui.html

### 本地开发

后端：

```bash
cd backend
mvn spring-boot:run
```

前端：

```bash
cd frontend
npm install
npm run dev
```

前端开发地址：http://localhost:5173
